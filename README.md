# QuizHub MVP (Vue + Node.js + Socket.IO)

MVP веб-приложения для проведения интерактивных квизов в реальном времени. Организатор создает квиз, запускает комнату, участники подключаются по коду и отвечают на вопросы во время live-сессии.

## Выбранный стек

- Frontend: `Vue 3`, `Vite`, `Pinia`, `Vue Router`, `Socket.IO Client`.
- Backend: `Node.js`, `Express`, `Socket.IO`.
- База данных: `SQLite` через `better-sqlite3`.
- Авторизация: `JWT` + `bcryptjs`.
- Контейнеризация: `Docker Compose`, nginx proxy для frontend/API/Socket.IO.

Почему так: стек полностью соответствует заданию по JavaScript fullstack, Socket.IO хорошо подходит для синхронных комнат, а SQLite удобна для учебного MVP без отдельного сервера БД.

## Что реализовано

- Регистрация и авторизация пользователей.
- Роли: `organizer` и `participant`.
- Конструктор квизов:
  - одиночный и множественный выбор;
  - вопросы с изображением по URL;
  - таймер и баллы на вопрос;
  - категория, описание, правила;
  - гибкие настройки проведения.
- Live-сессия по коду комнаты:
  - экран ожидания участников до старта;
  - старт, следующий вопрос, завершение;
  - доступ к ответу только во время активного вопроса;
  - обратный отсчет перед показом вопроса;
  - счетчик ответивших и не ответивших игроков;
  - автоматическое закрытие вопроса по таймеру;
  - realtime-события через Socket.IO;
  - live leaderboard.
- Режим администратора:
  - сводка по сессии с KPI;
  - график точности по вопросам;
  - распределение выбранных вариантов ответа;
  - график прогресса участников;
  - результаты каждого участника по вопросам;
  - экспорт результатов в CSV.
- Личный кабинет и история:
  - организатор видит созданные квизы и проведенные сессии;
  - участник видит историю участий и очки.

## Настройки квиза

По результатам анализа Kahoot, Wayground/Quizizz и Mentimeter в MVP добавлены:

- перемешивание порядка вопросов;
- перемешивание вариантов ответов;
- показ/скрытие лидерборда после каждого вопроса;
- показ/скрытие правильного ответа после вопроса;
- разрешение/запрет позднего входа;
- разрешение/запрет изменения ответа;
- максимум попыток на вопрос;
- обратный отсчет перед вопросом: `0`, `3`, `5`, `10`, `15`, `30` секунд;
- режимы очков:
  - `speed` — точность + скорость;
  - `accuracy` — только точность;
  - `disabled` — без очков.

## Docker-запуск

```bash

```

Открыть:

- приложение: [http://127.0.0.1:8080](http://127.0.0.1:8080)

Остановить:

```bash
docker compose down
```

```bash

```

## Тестирование

Backend:

```bash
cd /Users/dhelprat/Downloads/числаки/vk-quiz-mvp/backend
npm run smoke
npm run qa
npm audit
```

Frontend:

```bash
cd /Users/dhelprat/Downloads/числаки/vk-quiz-mvp/frontend
npm run build
npm run e2e
npm audit
```

`npm run qa` проверяет расширенные backend/API/WebSocket-сценарии: валидацию регистрации, роли, права доступа к квизам, аналитике и экспорту, запрет чтения правильных ответов участником, подключение по коду, экран ожидания, live-сессию из нескольких вопросов, прогресс ответивших, обратный отсчет, поздний вход, пустые и неверные ответы, запрет изменения ответа, лимит попыток, пересчет очков, историю и CORS.

`npm run e2e` проходит основной пользовательский сценарий в Chromium через Playwright: регистрация организатора, создание квиза с одиночным и множественным выбором, запуск комнаты, регистрация участника, вход по коду, экран ожидания, ответы, live-прогресс, завершение, просмотр режима администратора и скачивание CSV-результатов. Скриншоты успешного прогона сохраняются в `docs/qa-screenshots`.

## Критерии строгой проверки

Проект проверялся как учебная работа для защиты:

- полный сценарий должен проходиться без ручных исправлений данных;
- участник не должен видеть правильные ответы и чужую административную аналитику;
- ответы должны приниматься только во время активного вопроса;
- счетчик ответивших и обратный отсчет должны синхронно обновляться у ведущего и участников;
- экспорт должен быть доступен только организатору комнаты;
- лидерборд, история и аналитика должны сохраняться после завершения;
- режим администратора должен показывать KPI, графики точности, распределение ответов и прогресс участников;
- интерфейс должен быть понятным без инструкции;
- на основных страницах не должно быть наложений, слипаний текста и визуального мусора;
- сборка, audit, backend QA и browser e2e должны проходить без ошибок.
